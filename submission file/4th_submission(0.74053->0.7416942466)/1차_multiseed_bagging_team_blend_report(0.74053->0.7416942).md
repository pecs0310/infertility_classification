# 난임 분류 프로젝트 실험 보고서 - 1차 멀티시드 배깅 + 크로스팀 블렌딩

이 문서는 단일 시드(random_state)에 의한 우연성을 줄이기 위해 **멀티시드 풀파이프라인 배깅**을 적용하고, 팀원(v5)의 결과와 **크로스팀 블렌딩**을 시도한 1차 실험 보고서입니다.

---

## 1. 구현 사양 (Implementation Specification)

* **목적**: 단일 random_state에 따른 노이즈을 줄이고, 서로 다른 팀원 파이프라인을 결합해 일반화 성능을 추가로 끌어올림.
* **폴더 경로**: `experiment_history/1차/`
* **검증 산출물**: `experiment_history/submit file/` (OOF 비교용 npy 캐시 포함)
* **실제 제출 산출물**:
  - `submission file/1차_3model_bagged_ensemble_submit(0.74053->0.7416942466).csv` — **제출 완료**
  - `submission file/1차_final_team_blend_submit(0.74071->).csv` — **제출 후보 (미제출)**

---

## 2. 핵심 기법 (Methodology)

### 2.1 멀티시드 풀파이프라인 배깅 (Multi-seed Bagging)

* **방법**: `StratifiedKFold`와 각 모델의 `random_state`를 10가지 값(1004, 42, 7, 123, 2024, 88, 555, 999, 31, 77)으로 바꿔서 전체 5-Fold 파이프라인을 10회 반복. 10개의 test 예측을 평균.
* **근거**: 단일 시드 OOF 점수의 표준편차를 직접 측정(0.00011)하여, 시드 변동 자체가 "노이즈"의 실측 크기임을 확인. 이를 평균내면 운에 덜 좌우되는 추정치를 얻을 수 있음.
* **적용 모델**: LightGBM, CatBoost, XGBoost 3종 각각에 동일하게 적용.

### 2.2 우리팀 3종 가중평균 앙상블

* LightGBM(배깅) + CatBoost(배깅) + XGBoost(배깅)을 0.1 단위 그리드서치로 최적 가중치 탐색.

### 2.3 크로스팀 블렌딩

* 우리팀 3종 배깅 앙상블과 김영혜님 v5 앙상블(LightGBM+XGBoost+CatBoost, 0.4/0.3/0.3)을 같은 train 행 순서(y값 일치 확인 완료, 차이 0건) 기준으로 블렌딩.

## 4. 결론

- 멀티시드 배깅은 **모든 모델에서 일관되게 작은 개선**을 만들어낸, 지금까지 시도한 기법 중 가장 신뢰도 높은 레버였습니다 (피처 엔지니어링이나 모델 다양화와 달리, 시드별 표준편차로 효과를 직접 실측 가능).
- **흥미로운 발견**: 완전히 독립적으로 만들어진 두 파이프라인(우리팀 3종 배깅 vs 김영혜님 v5)이 **실제 리더보드에서 거의 동일한 지점(0.74169 vs 0.74173, 차이 0.00004)**에 도달했습니다. 이는 이 데이터셋의 실질적인 성능 한계가 약 0.7417 부근이라는 가설을 실제 제출 결과로 한 번 더 뒷받침합니다.
- 두 파이프라인을 블렌딩한 결과(OOF 0.74071)는 로컬 기준으로는 가장 높은 점수이며, 아직 실제 제출 검증 전입니다.
