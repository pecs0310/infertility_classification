# 난임 분류 프로젝트 실험 보고서 - 2차 풍부한 피처셋 + ECDF 랭크 스태킹

이 문서는 1차 멀티시드 배깅 + 크로스팀 블렌딩(OOF 0.74071) 이후, **풍부한 피처 엔지니어링을 적용한 새 모델 후보들을 추가**하고, **ECDF 랭크변환 + 선형회귀 메타러너 스태킹**으로 기존 12개 모델 후보를 결합한 2차 실험 보고서입니다.

---

## 1. 구현 사양 (Implementation Specification)

* **목적**: 1차에서 확정된 0.74071(우리팀 3종 배깅 + v5 가중평균)을 기준선으로 삼아, (1) 더 정교한 피처 엔지니어링을 적용한 새 모델을 추가하고, (2) 단순 가중평균보다 유연한 스태킹 메커니즘으로 기존 후보들을 재조합하여 추가 개선을 시도함.
* **폴더 경로**: `experiment_history/2차/`
* **검증 산출물**: `experiment_history/blend_cache/` (모델별 OOF/test 예측 npy 캐시)
* **실제 제출 산출물**:
  - `submission file/2차_ecdf_stack_local(0.74098->0.7421890122).csv` — **제출 완료**

## 2. 핵심 기법 (Methodology)

### 2.1 풍부한 피처 엔지니어링 (Rich Feature Set)

기존 베이스라인 전처리(결측치 `-1`/`해당없음` 일괄 대치)에서 한 단계 더 나아가, 다음 요소들을 추가한 피처셋을 설계함:

* **Keep-NaN 정책**: DI 시술의 구조적 결측(배아/난자 관련 컬럼)을 `-1`로 채우지 않고 진짜 결측으로 유지하며, `_missing`/`_not_applicable_DI` 플래그를 별도 생성. 트리 모델의 네이티브 결측치 분기 학습을 활용.
* **아웃라이어 클리핑**: 배아/난자 개수 컬럼에 상한선을 적용하고(`총 생성 배아 수→40` 등) `_high_flag`로 원래 값이 상한을 넘었는지 별도 표시.
* **안전한 비율(ratio) 피처**: 분모가 0이거나 결측이면 `0`으로 임의 대치하는 대신 진짜 NaN + `_available` 플래그로 분리(`fertilization_rate`, `embryo_utilization_rate` 등 6종).
* **실효 모성 나이(effective maternal age)**: 기증 난자 사용 시 환자 본인 나이 대신 기증자 나이 중간값으로 대체한 연속형 피처. 환자 나이와 기증자 나이의 차이(`donor_rejuvenation_gap`)도 별도 생성.
* **시술 유형 토큰화**: `특정 시술 유형` 텍스트 컬럼에서 IVF/ICSI/IUI/FER/BLASTOCYST 등 하위 토큰을 추출해 binary 플래그로 분리.
* **네이티브 categorical**: LabelEncoder 정수 인코딩 대신 `pd.Categorical` dtype으로 CatBoost/LightGBM의 진짜 범주형 분할 알고리즘을 사용 (train 카테고리만 사용, test 전용 신규 값은 sentinel로 흡수 — 대회 규칙 준수).

### 2.2 풍부한 피처셋 기반 신규 모델 3종

| 모델명 | 설명 |
|---|---|
| `donor_cat` | CatBoost + 풍부한 피처셋, 7시드 평균(멀티시드 배깅 적용) |
| `alpha_cat` | CatBoost + 풍부한 피처셋 + `class_weights=[0.6, 0.4]` (약한 재가중) |
| `lgbm_spw2_richfeat` | LightGBM + 풍부한 피처셋 + `scale_pos_weight=2.0` |

### 2.3 ECDF 랭크변환 + 선형회귀 메타러너 스태킹

기존 가중평균 방식(가중치 0~1, 합=1 제약)의 한계를 보완하기 위해 새 스태킹 메커니즘을 도입:

* 각 fold의 **train 부분에서만** ECDF(경험적 누적분포함수)를 적합하여, 모든 후보 모델의 OOF/test 예측을 동일한 백분위 스케일로 정규화 (fold-safe, 누수 없음).
* 정규화된 예측들을 **선형회귀(LinearRegression)**로 결합 — 가중평균과 달리 **음수 계수를 허용**하여 모델간 공유된 노이즈를 상쇄할 수 있음.
* 비교 검증을 위해 **Ridge(L2 정규화) 메타러너**로도 교체하여 alpha 0~30 그리드 테스트 — 결과가 전 구간에서 거의 동일(0.74097~0.74098)하게 나와 LinearRegression 결과가 다중공선성에 의한 우연이 아님을 확인.
* 추가로 **Greedy Ensemble Selection**(한 번에 하나씩, 개선될 때만 채택)도 별도 구현해 교차 검증 — 전체 12개 동시 회귀(0.74095~0.74098)보다 다소 보수적인 값(0.74076~0.74088)을 보였으나, 동일한 방향성을 재확인함.

### 2.4 팀원 파이프라인 규칙 준수 수정

블렌딩 후보로 사용된 팀원(김영혜)의 v5 앙상블 파이프라인에서, 범주형 인코딩 시 `train ∪ test` 합집합으로 카테고리를 구성하던 부분을 발견하여 **train 전용 카테고리 + 미관측값 fallback** 방식으로 수정함 (DACON 규칙 "label/one-hot encoding 시 test 데이터셋 활용" 위반 해소). 수정 전후 점수 차이는 +0.00001로 실질적 영향 없음을 확인 후 교체.

## 3. 모델 개선 및 검증 결과 (Model & Performance)

### 3.1 개별 모델 OOF 비교 (전체 12개 후보)

| 모델 | OOF AUC |
|---|---|
| `donor_cat` (CatBoost, 7시드, 풍부피처셋) | **0.74076** |
| `v5_ensemble` (팀원, 규칙 준수 수정판) | 0.74043 |
| `10seed_bagged` (LightGBM, 1차 멀티시드 배깅) | 0.74036 |
| `alpha_cat` (CatBoost, class_weights) | 0.74033 |
| `xgboost_bagged` | 0.74010 |
| `catboost_bagged` | 0.74005 |
| `feature_subspace` (피처 서브샘플 배깅) | 0.73973 |
| `xgb_rankpairwise` | 0.73939 |
| `lgbm_lambdarank` | 0.73748 |
| `lgbm_keepnan` (LightGBM, Keep-NaN만 단독 적용) | 0.73648 |
| `mlp` | 0.73313 |
| `lgbm_spw2_richfeat` | 0.73297 |

→ `donor_cat`이 단독 모델 중 최고 성능을 기록. 풍부한 피처셋과 CatBoost의 조합이 다른 모델 다양화 시도(EBM, MLP, rank-loss 계열)보다 효과적이었음.

### 3.2 클리핑 효과 단독 검증

`donor_cat`과 동일 설정으로 클리핑 유무만 비교(단일시드): 클리핑 있음 0.74047 vs 없음 0.74037, 차이 +0.00009로 노이즈 수준. 클리핑은 유지하되, 있고 없고가 실질적 차이를 만들지 않음을 확인.

### 3.3 ECDF 랭크 스태킹 최종 결과

* **전체 12개 후보 → ECDF + LinearRegression 스택 OOF**: **0.74098**
* 채택된 조합은 `donor_cat`, `v5_ensemble`을 포함한 전체 후보의 가중 결합 (음수 계수 포함)
* Ridge(alpha 0~30) 재검증: 전 구간 0.74097~0.74098로 안정적 — 결과의 신뢰도 확인

### 3.4 종합 비교표

| 항목 | OOF | 실제 제출 | OOF→실제 격차 |
|---|---|---|---|
| 팀원의 v5 앙상블 | 0.74043 | 0.7417306379 | +0.00130 |
| 우리팀 3종 배깅 앙상블 | 0.74053 | 0.7416942466 | +0.00116 |
| 우리팀 + v5 가중평균 블렌딩 (1차 최종) | 0.74071 | 0.7418495884 | +0.00114 |
| **풍부한 피처셋 CatBoost 등 12종 → ECDF 랭크 스택 (2차 최종)** | **0.74098** | **0.7421890122** | **+0.00121** |

→ OOF와 실제 점수의 격차가 4건 모두 **+0.0011~0.0013 범위로 매우 일관적**임을 재확인. 이 안정적인 격차 패턴 덕분에 로컬 검증만으로 실제 제출 점수를 신뢰성 있게 예측할 수 있었음.

## 4. 결론

- **풍부한 피처 엔지니어링(Keep-NaN, 클리핑, 안전한 비율, 실효나이) + CatBoost** 조합이 1차에서 확보한 모델들보다 더 강한 단독 모델(`donor_cat`, OOF 0.74076)을 만들어냈고, 이것이 2차 개선의 핵심 동력이었음.
- **ECDF 랭크변환 + 선형회귀 메타러너**는 기존 가중평균 대비 유연한 결합(음수 계수 허용)을 제공했으며, Ridge 재검증으로 결과의 안정성을 확인함.
- **`donor_cat`의 시드를 3개→7개로 늘렸을 때 개선은 +0.00003에 불과** — 멀티시드 배깅의 수확체감 구간에 진입했음을 확인. 이후 추가 시드 투자보다 새로운 모델 후보 발굴이 더 효율적인 시점으로 판단.
- **클리핑 등 세부 전처리 옵션은 노이즈 수준의 영향**만 있었으며, 핵심은 Keep-NaN + 안전한 비율 + 실효나이 같은 구조적 변경과 모델(CatBoost) 선택이었음.
- 최종적으로 **OOF 0.74098 → 실제 0.7421890122**를 달성, 1차 0.74071(실제 0.7418496) 대비 실제 점수 기준 **+0.00034 개선**. 리더보드 6위 → 3위로 상승.
- OOF→실제 점수 격차가 4건의 제출 모두 +0.0011~0.0013 범위에서 일관되게 나타나, 이 프로젝트 전반에 걸쳐 **로컬 nested CV 검증 결과를 실제 순위 예측에 신뢰성 있게 활용**할 수 있었음.
