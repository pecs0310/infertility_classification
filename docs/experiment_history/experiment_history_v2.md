## 1차 — TabTransformer + LightGBM 블렌딩 (커널 크래시 디버깅 포함)

- **문제**: 노트북이 자꾸 멈추거나 커널이 죽음 (40분+ 대기 후 무응답)
- **진짜 원인**: torch와 LightGBM을 같은 프로세스에서 같이 쓸 때 **스레드 풀 충돌(데드락)** 발생
- **해결**: `torch.set_num_threads(1)`로 1차 완화 → 근본적으로는 **LightGBM과 TabTransformer를 완전히 분리된 노트북(별도 프로세스)**으로 나눠서 해결
- **결과**: LightGBM 0.73925 / TabTransformer 0.73523 / 상관계수 0.9686
- **블렌딩 효과**: +0.00047 → **노이즈 수준**
- **교훈**: 무거운 라이브러리(torch+lightgbm/xgboost/catboost)는 절대 같은 커널에서 같이 돌리지 말 것 → 이후 모든 노트북을 "기능별 분리" 패턴으로 설계

---

## 2차 — ROC-AUC 채점 방식 이해 (사이드 에피소드)

- sample_submission(전부 0.0)을 제출해서 정확히 0.5 확인
- AUC는 임계값과 무관하게 "양성-음성 쌍 비교"로 채점된다는 것, 동점이면 항상 0.5가 나온다는 것을 직접 실증
- 이후 모든 실험에서 "이게 진짜 신호인지 노이즈인지" 판단할 때 이 이해가 기반이 됨

---

## 3차 — CatBoost 재튜닝 (Optuna 30trial + 3fold → 5fold)

- **블렌딩**(LightGBM 미튜닝본과): 상관계수 0.9912, 개선 +0.00031 → **노이즈**

## 4차 — 결측치 정밀 대치 실험 (배아 이식 경과일 / 기증자 나이 진짜 결측 분리)

- 팀원 Antigravity 분석 기반: "이식했지만 날짜 누락(진짜결측, 740건)"을 구조적결측(-1)과 분리해서 임상 최빈값으로 대치
- **결과**: -0.00022 (오히려 손해) → **효과 없음, 영향받는 행이 전체의 0.3%뿐이라 너무 적음**

## 9차 — 연령대별 Local AUC 분석 + 나이분리 서브모델 실험

- 팀원 분석("고령층 FNR 47~68%")을 직접 검증
- **발견**: 고령층(43-50세) Local AUC가 오히려 가장 높음(0.83+) — FNR은 그냥 "원래 성공률이 낮아서 임계값 기준으로만 그렇게 보이는 것"이었음
- **진짜 약점**: 젊은층(18-37세, 전체 62%)의 Local AUC가 가장 낮음(0.70~0.71)
- 만37세 기준 분리 모델 실험 → 전체 -0.00076, 젊은층 구간만 봐도 -0.00055 (**오히려 손해**)
- **결론**: 통합 모델이 이미 나이 정보를 알아서 잘 활용하고 있음, 수동 분리는 항상 손해라는 패턴 재확인
