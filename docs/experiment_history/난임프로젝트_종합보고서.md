# 난임 시술 임신 성공 예측 — 프로젝트 종합 보고서

---

## 1. 프로젝트 개요

* **과제**: 난임 시술(IVF/DI) 환자의 시술 데이터를 바탕으로 임신 성공 여부를 예측
* **평가지표**: ROC-AUC (Public 100% 반영, 최종 점수와 동일 — Private 분리 없음)
* **데이터 규모**: train 256,351행 / test 90,067행, 피처 67개
* **타깃 분포**: 임신 성공 25.83% / 실패 74.17% (중간 정도 불균형)
* **최종 성과**: **OOF 0.74098 → 실제 제출 0.7421890122 (Public 리더보드 3위)**

---

## 2. 데이터 이해 (EDA 핵심 발견)

| 발견 | 내용 |
|---|---|
| 나이 | 가장 강력한 단일 변수. 만18-34세 32.3% → 만43-44세 11.8% |
| 시술 완료 여부 | 이식된 배아 수=0 → 성공률 0.08% (전체 14.6%, "시술 미완료" 신호) |
| 배아 이식 시점 | 이식 경과일=5일(배반포 이식) → 성공률 40.4%, 가장 높은 단일 구간 |
| 시술 유형 | IVF 26.2% vs DI 12.9%, 고령일수록 격차 확대 |
| 구조적 결측 | 배아/난자 관련 컬럼 결측 6,291건 = DI 시술자 전원과 정확히 일치 |
| **난자 출처 × 나이 (심슨의 역설)** | 본인 난자는 나이 들수록 32.6%→2.8%로 붕괴하지만, **기증 난자는 나이와 거의 무관하게 29~34% 유지**. 45-50세 그룹의 성공률이 다시 튀어오르는 것처럼 보이는 이유는 이 그룹의 기증 난자 비율이 53%까지 치솟기 때문 |
| Local AUC 분석 | 고령층(43-50세) 내부 판별력이 오히려 가장 높음(0.83+/0.75+). 모델이 "고령층을 못 맞춘다"는 인상은 **저위험군 기준 전역 임계값을 고위험군(낮은 base rate)에 적용했을 때 생기는 통계적 착시**이며, ROC-AUC(임계값 비의존)에는 해당 사항 없음 |
| 환자 단위 반복 구조 | train 내 일부 행이 통계적으로 유의하게 반복 패턴을 보이나, 전체 피처 기준 완전 일치 행은 0건 — 직접 익스플로잇 불가 |

## 3. 전처리 / 피처 엔지니어링

### 3.1 기본 전처리
* 구조적 결측 플래그(`is_DI`, `froze_embryo`) 생성
* 정보량 없는 컬럼 2개 제거(`불임 원인-여성요인`, `난자 채취 경과일`)
* 시술/임신/출산 횟수 등 순서형 텍스트를 정수로 매핑

### 3.2 최종 채택 — 풍부한 피처 엔지니어링
초기에는 결측치를 `-1`/`해당없음`으로 일괄 대치했으나, 최종 단계에서 다음으로 고도화:

| 요소 | 설명 |
|---|---|
| Keep-NaN 정책 | DI의 구조적 결측을 `-1`로 채우지 않고 진짜 NaN으로 유지 + `_missing`/`_not_applicable_DI` 플래그. 트리 모델의 네이티브 결측 분기 학습 활용 |
| 아웃라이어 클리핑 | 배아/난자 개수 컬럼 상한 적용 + `_high_flag`로 원래 초과 여부 별도 표시 |
| 안전한 비율(ratio) 피처 | 분모가 0/결측이면 임의의 0이 아니라 진짜 NaN + `_available` 플래그로 분리(수정률/배아활용률 등 6종) |
| 실효 모성 나이 | 기증 난자 사용 시 환자 본인 나이 대신 기증자 나이 중간값으로 대체한 연속형 피처 + 환자-기증자 나이 격차 피처 |
| 시술유형 토큰화 | `특정 시술 유형` 텍스트에서 IVF/ICSI/IUI/FER/BLASTOCYST 등 하위 토큰을 binary 플래그로 분리 |
| 네이티브 categorical | LabelEncoder 정수 인코딩 대신 `pd.Categorical` dtype으로 CatBoost/LightGBM의 진짜 범주형 분할 알고리즘 사용 |

이 피처셋을 적용한 **CatBoost(7시드 배깅)**가 프로젝트 전체에서 가장 강력한 단독 모델(OOF 0.74076)이 되었음.

### 3.3 검증되었으나 채택하지 않은 피처 시도
교차피처, 비율형 피처(단순 버전), 클러스터링 기반 피처, 인터랙션 카테고리 결합, 환자그룹
target encoding 등 다수를 시도했으나 전부 노이즈 수준이거나 손해로 확인됨(상세는 실험 로그 참고).

## 4. 모델링 전략

### 4.1 멀티시드 풀파이프라인 배깅 — 핵심 레버
* `StratifiedKFold` + 모델 `random_state`를 여러 값으로 바꿔 전체 파이프라인을 반복·평균
* 시드별 OOF 점수의 **표준편차를 0.00011로 직접 실측**, 이를 "노이즈 크기"의 기준으로 프로젝트 전체에서 사용
* 모든 모델(LightGBM, CatBoost, XGBoost, donor_cat 등)에서 일관되게 **+0.0003~0.0004** 개선
* 시드를 늘릴수록 개선은 수확체감(예: donor_cat 3→7시드는 +0.00003에 불과)

### 4.2 모델 후보 풀 (최종 12종)
| 모델 | OOF AUC | 비고 |
|---|---|---|
| `donor_cat` | 0.74076 | CatBoost + 풍부한 피처셋, 7시드 배깅 — **단독 최고** |
| `v5_ensemble` | 0.74043 | 팀원 파이프라인(LightGBM+XGBoost+CatBoost), 컴플라이언스 수정판 |
| `10seed_bagged` | 0.74036 | LightGBM, 1차 멀티시드 배깅 |
| `alpha_cat` | 0.74033 | CatBoost + class_weights 약한 재가중 |
| `xgboost_bagged` / `catboost_bagged` | 0.74010 / 0.74005 | 1차 3종 배깅의 나머지 두 모델 |
| `feature_subspace` | 0.73973 | 피처 90% 무작위 샘플링 10개 모델 배깅 |
| `xgb_rankpairwise` / `lgbm_lambdarank` | 0.73939 / 0.73748 | 순위 손실함수 기반 |
| `lgbm_keepnan` / `mlp` / `lgbm_spw2_richfeat` | 0.73648 / 0.73313 / 0.73297 | 모델 다양성 보강용, 단독 성능은 낮음 |

### 4.3 모델 다양성에 대한 핵심 발견
XGBoost, CatBoost, 로지스틱회귀, MLP, ExtraTrees, TabTransformer, EBM, 순위손실 모델 등을
폭넓게 시도했으나, **거의 모든 모델이 LightGBM 기본 모델과 OOF 상관계수 0.95~0.99**를 보임.
즉 알고리즘을 바꿔도 비슷한 신호로 수렴하는 경향이 강했고, 진짜 의미 있는 차별화는 **피처
엔지니어링 수준의 변화**(풍부한 피처셋)에서 나왔음.

## 5. 앙상블 / 스태킹 아키텍처

### 5.1 단순 가중평균 (1차 최종)
우리팀 3종 배깅 앙상블 + 팀원 v5 앙상블을 0.6:0.4로 가중평균 → OOF 0.74071

### 5.2 ECDF 랭크변환 + 선형회귀 메타러너 (2차 최종, 채택)
* 각 fold의 **train 부분에서만** ECDF(경험적 누적분포함수)를 적합하여 모든 후보의 예측을
  동일한 백분위 스케일로 정규화 (fold-safe, 데이터 누수 없음)
* 정규화된 예측들을 **선형회귀**로 결합 — 가중평균(0~1, 합=1 제약)과 달리 **음수 계수를
  허용**하여 모델간 공유 노이즈를 상쇄
* **Ridge(L2) 메타러너로 alpha 0~30 재검증** → 결과가 전 구간에서 거의 동일(0.74097~0.74098)
  하게 나와, LinearRegression 결과가 다중공선성에 의한 불안정한 우연이 아님을 확인
* **Greedy Ensemble Selection**(한 번에 하나씩, 개선될 때만 채택)으로도 별도 교차검증 —
  전체 동시 회귀보다 다소 보수적이지만 같은 방향성 재확인

→ **최종: 12개 모델 → ECDF + 선형회귀 스택 = OOF 0.74098**

## 6. 검증 방법론

* **모든 인코딩/스케일링/결측치 처리는 train 데이터에만 fit**, test 데이터의 분포·통계는
  어떤 단계에서도 모델 구조 결정에 사용하지 않음 (컴플라이언스 7번 항목 참고)
* **Nested CV**: 가중치/메타러너 학습은 fold의 train 부분에서만, 평가는 held-out fold로
* **OOF AUC ↔ 실제 제출 점수 격차를 4건의 제출 전부에서 추적**: +0.00114 ~ +0.00130 사이로
  매우 일관적 → 이 패턴 덕분에 로컬 검증만으로 실제 순위 변화를 신뢰성 있게 예측 가능했음

| 제출 | OOF | 실제 | 격차 |
|---|---|---|---|
| 팀원 v5 단독 | 0.74043 | 0.7417306379 | +0.00130 |
| 우리팀 3종 배깅 | 0.74053 | 0.7416942466 | +0.00116 |
| 1차 최종 블렌딩 | 0.74071 | 0.7418495884 | +0.00114 |
| **2차 최종 (ECDF 스택)** | **0.74098** | **0.7421890122** | **+0.00121** |

## 7. 컴플라이언스 관리

대회 규칙(평가 데이터셋을 모델 학습/인코딩/스케일링/결측치 통계에 활용 시 Data Leakage로
수상 제외)을 프로젝트 전반에서 점검:

* **XGBoost 네이티브 categorical**: `train ∪ test` 합집합으로 카테고리를 구성하던 위반을
  발견·수정 (train 카테고리만 사용, test 신규값은 결측 처리). 영향 행 3/90,067건, 점수 영향 없음
* **팀원 v5 파이프라인**: 동일 패턴의 위반을 발견·수정 (train 카테고리 + 최빈값 fallback).
  영향 행 0건, 점수 영향 +0.00001
* **ECDF 메타러너**: fold-train에서만 분포를 적합하도록 설계하여 처음부터 누수 차단
* **Pseudo-labeling**: test 데이터의 행(피처)이 학습 데이터에 포함되는 기법 특성상 규칙
  회색지대로 판단, **운영진에게 직접 문의하여 답변 전까지 제출 보류** 중

## 8. 핵심 인사이트

1. **확실한 신호는 "더 많은 모델"이 아니라 "더 좋은 피처 + 안정적인 배깅"에서 나왔다.**
   알고리즘 다양화는 대부분 비슷한 결론(상관계수 0.96+)으로 수렴했지만, 결측치/비율 처리를
   정교화한 피처셋은 실질적 개선을 만들어냈다.
2. **부분집단·클래스 단위 재가중 계열은 이 데이터셋+ROC-AUC 조합에서 구조적으로 역효과였다.**
   재가중 강도에 비례해 단조적으로 성능이 떨어지는 패턴이 여러 독립적인 기법에서 반복 관찰됨 —
   트리 기반 모델이 이미 데이터에서 최적 분할을 찾은 상태이므로, 외부에서 가중치를 흔들면
   그 균형이 깨지는 것으로 해석된다.
3. **"FNR이 높다"는 진단은 ROC-AUC 평가에서 함정이 될 수 있다.** 저위험군 기준 임계값을
   고위험군(낮은 base rate)에 적용하면 FNR이 치솟지만, 같은 구간의 Local AUC(순위 판별력)는
   오히려 더 높을 수 있다 — 이 프로젝트에서 두 번 독립적으로 확인됨.
4. **블렌딩 메커니즘보다 "어떤 모델을 모았는지"가 더 중요했다.** 가중평균·rank·isotonic·전역
   최적화·greedy·Ridge 등 다른 방법론들이 결국 0.7406~0.7410의 좁은 범위로 수렴.
5. **OOF-실제 점수 격차의 일관성**(+0.0011~0.0013)이 이 프로젝트의 의사결정을 처음부터
   끝까지 뒷받침한 가장 중요한 검증 도구였다.
6. **회색지대 기법은 효과가 보여도 운영진 확인 전까지 보류하는 것이 안전하다.**
