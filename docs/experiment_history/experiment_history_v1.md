## 2. 효과가 없었던 시도들 (전부 검증함, 반영 안 함)

### 피처 엔지니어링
- **교차 피처 5종** (나이×이식배아수, 나이×시술유형, 나이×시술횟수 등): 전부 노이즈 수준(±0.0001), random_state 바꾸면 부호도 뒤집힘
- **비율형 피처** (배아전환율, 저장율, 이식율): 오히려 일관되게 손해 (-0.0003~0.0004)
- **시술유형 복합카테고리 분해** (`특정 시술 유형`의 ICSI/BLASTOCYST/AH 플래그 분리): 효과 없음
- **결측 패턴 조합** (`stage` 변수, `missing_count`): 효과 없음 — 원본 컬럼들에 이미 정보가 내재되어 있었음
- **타겟 인코딩** (LabelEncoder 대체): -0.00016
- **나이/시술횟수 순서형 숫자 직접 매핑**: 효과 없음
- **클러스터링 기반 서브그룹 피처** (KMeans k=5/8/12): 전부 노이즈~약한 손해

**핵심 인사이트**: LightGBM은 트리 분기를 통해 이런 조합을 스스로 학습할 수 있어서, 사람이 미리 만들어주는 명시적 피처는 거의 항상 중복 정보가 됨.

### 모델 구조
- **모델 다양화 (앙상블)**: XGBoost(0.736), CatBoost(0.737), 로지스틱회귀(0.7235), MLP(0.7296), ExtraTrees(0.7283) — 전부 LightGBM과 OOF 예측 상관계수 0.91~0.98로 매우 높아 앙상블 효과 없음
- **TabTransformer (딥러닝, 임베딩+Self-Attention)**: 0.7362 (15 epoch) — LightGBM 못 미침. 다만 LightGBM과의 OOF 상관계수가 0.95로 다른 트리 모델보다 낮아서 블렌딩 가능성은 남아있음 (검증 중)
- **클래스 불균형 처리** (`scale_pos_weight`, `is_unbalance`): 둘 다 손해 (-0.0003). ROC-AUC는 임계값 독립적 지표라 불균형 보정 효과가 적음
- **부스팅 타입 변경** (`dart`, `goss`): 둘 다 기본 `gbdt`보다 나쁨
- **IVF/DI 서브모델 분리**: 전체 손해, 특히 DI(전체의 2.5%, 6,291건)는 표본 부족으로 큰 손해 (0.681→0.659)
- **시드 앙상블** (같은 파라미터, 다른 random_state 3개 평균): 빠른 검증에서는 미세 개선(+0.0001~0.0002) 봤으나, 실제 5-Fold 전체 적용 시 0.74002로 튜닝 단독과 동일 — 노이즈였던 것으로 결론
- **K-Fold 수 증가** (5→10): random_state 3개로 검증 시 +0.00024/-0.00006/+0.00007로 부호 불일치 — 노이즈
- **Pseudo-labeling** (test 확신도 높은 예측을 학습에 추가): -0.00023, 효과 없음. 참고로 "확실한 성공(0.97+)" 케이스가 0건 발견됨 (test 예측 최대값 0.70)

## 3. 데이터 자체에 대한 핵심 발견 (EDA)

- **나이가 가장 강력한 단일 변수**: 만18-34세 32.3% → 만43-44세 11.8% 성공률
- **이식된 배아 수 = 0 → 성공률 0.08%**: 사실상 "시술 미완료" 신호, 전체의 약 14.6%(37,284건)
- **시술 유형(IVF 26.2% vs DI 12.9%)**: 고령일수록 격차 더 벌어짐 (DI는 고령에서 거의 무력화)
- **배아 이식 경과일 = 5일(배반포 이식) → 성공률 40.4%**: 가장 높은 단일 구간, 임상적으로도 알려진 패턴
- **구조적 결측**: 배아/난자 수치 결측 6,291건 = DI 시술자 전원과 정확히 일치 ("해당없음"이지 "정보없음"이 아님)
- **시술 시기 코드**: 익명화된 코드지만 나이/시술유형과 독립적인 성공률 차이(24.5~26.9%)를 가짐 — 진짜 신호, 이미 모델이 활용 중
- **강한 상호작용 존재 확인**: 기증 난자(만21-25세) + 5일 이식 조합 시 49.2% (단순 합보다 높음) — 다만 명시적 피처로 추가해도 모델 성능 개선 없음 (이미 원본 컬럼으로 학습 가능)

## 7. 종합 결론

지금까지 시도한 거의 모든 방향(피처 엔지니어링 9종, 모델 구조 변경 8종)이 효과가 없거나 미미했습니다. 이는 현재 피처셋과 LightGBM 조합이 **데이터가 가진 정보량의 한계에 가깝게 도달했다는 신호**로 보입니다. 다른 팀의 0.742 점수와의 격차(약 0.002~0.0024)는 아직 명확히 설명되지 않았으며, 운영진 규칙 확인 결과 외부 데이터나 명백한 규칙 위반 가능성은 낮아 보입니다.

남은 유력 후보는 LightGBM-TabTransformer 블렌딩과 CatBoost 재튜닝이며, 둘 다 검증 중입니다.
