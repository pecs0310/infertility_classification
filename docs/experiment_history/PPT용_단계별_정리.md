# 난임 시술 임신 성공 예측 — 단계별 정리 (PPT용)

**최종 결과: OOF 0.74098 → 실제 0.7421890122 (Public 3위)**

---

## 1. EDA

**평가지표 / 데이터**
- ROC-AUC, Public 100% 반영(Private 분리 없음)
- train 256,351행 / test 90,067행, 피처 67개
- 타깃 분포: 성공 25.83% / 실패 74.17%

**핵심 발견**
- **나이**가 가장 강력한 단일 변수: 만18-34세 32.3% → 만43-44세 11.8%
- **이식된 배아 수=0 → 성공률 0.08%** ("시술 미완료" 신호, 전체 14.6%)
- **배아 이식 경과일=5일(배반포 이식) → 성공률 40.4%**, 단일 구간 중 최고
- 시술 유형: IVF 26.2% vs DI 12.9%, 고령일수록 격차 확대
- 구조적 결측(배아/난자 컬럼) 6,291건 = **DI 시술자 전원과 정확히 일치**
- **난자 출처 × 나이 상호작용(심슨의 역설)**: 본인 난자는 나이 들수록 32.6%→2.8%로 붕괴, **기증 난자는 나이와 무관하게 29~34% 유지** → 45-50세 그룹 성공률이 다시 튀어오르는 건 기증 비율이 53%까지 늘어나기 때문
- Local AUC 분석: "고위험군은 모델이 못 맞춘다"는 인상은 **임계값 기준 통계적 착시**(저위험군 기준 임계값을 낮은 base-rate 그룹에 적용한 결과) — 실제 그룹 내부 판별력은 양호

## 2. 전처리

- 핵심 버그 수정: `predict()` → `predict_proba()` (AUC 계산 정상화, 가장 큰 단일 개선)
- `stratify=y` 적용 (클래스 비율 유지)
- 구조적 결측 플래그 생성: `is_DI`, `froze_embryo`
- 정보량 없는 컬럼 2개 제거
- 검증 체계: 단일 split → **Stratified 5-Fold**로 전환 (안정성 확보)
- 범주형 인코딩: LabelEncoder / 네이티브 `pd.Categorical` dtype 두 가지 병행 검증
- **컴플라이언스**: 범주형 인코딩 시 `train ∪ test` 합집합으로 카테고리를 구성하던 위반 2건(XGBoost 자체 파이프라인, 팀원 파이프라인) 발견·수정 → **train 전용 카테고리 + 미관측값 fallback** 방식으로 표준화

## 3. 피처 엔지니어링 (개요)

**핵심 인사이트**: 트리 기반 모델은 변수 간 상호작용을 분기로 스스로 학습하기 때문에, 사람이
미리 만들어주는 명시적 피처는 대부분 **중복 정보**가 됨.

**효과 없었던 시도 (전부 검증 완료)**
- 교차 피처(나이×배아수 등), 단순 비율형 피처, 클러스터링 기반 피처(KMeans)
- 인터랙션 카테고리 결합, 타겟 인코딩, 환자그룹 target encoding(특히 손해 큼)

**효과 있었던 방향**
- 결측치를 "채우는" 게 아니라 "구조적 의미를 보존"하는 방식(Keep-NaN) + 비율 피처를 "정확하게"(분모 0/결측 구분) 만든 것 → 자세한 목록은 4번(파생피처) 참고

## 5. 파생 피처 (최종 채택 — 풍부한 피처셋)

| 피처 | 설명 |
|---|---|
| Keep-NaN 정책 | DI 구조적 결측을 `-1`로 채우지 않고 진짜 NaN 유지 + `_missing`/`_not_applicable_DI` 플래그 |
| 아웃라이어 클리핑 | 배아/난자 개수 상한 + `_high_flag` |
| 안전한 비율(ratio) 피처 6종 | 분모 0/결측 시 진짜 NaN + `_available` 플래그 (수정률, 배아활용률 등) |
| **실효 모성 나이** | 기증 난자 사용 시 환자 나이 대신 기증자 나이로 대체 + `donor_rejuvenation_gap` |
| 시술유형 토큰화 | IVF/ICSI/IUI/FER/BLASTOCYST 등 하위 토큰 binary 분리 |
| 네이티브 categorical | `pd.Categorical` dtype으로 진짜 범주형 분할 알고리즘 사용 |

→ 67개 → **145개** 피처로 확장, 이 피처셋 + CatBoost 조합이 단독 모델 최고 성능 견인

## 6. 모델링

**핵심 레버 — 멀티시드 풀파이프라인 배깅**
- `random_state`를 여러 값으로 바꿔 전체 파이프라인 반복·평균
- 시드별 표준편차 **0.00011**을 직접 실측, 이후 모든 실험의 "노이즈 판단 기준"으로 사용
- 모든 모델에서 일관되게 **+0.0003~0.0004** 개선

**모델 후보 풀 (최종 12~13종)**

|---|---|
| `donor_cat` (CatBoost+풍부한피처셋, 7시드) | 0.74076 |
| `v5_ensemble` (팀원) | 0.74043 |
| `10seed_bagged` (LightGBM) | 0.74036 |

**모델 다양성에 대한 핵심 발견**: XGBoost/CatBoost/로지스틱/MLP/ExtraTrees/TabTransformer/EBM/순위손실 모델 등 폭넓게 시도했으나 **대부분 LightGBM과 상관계수 0.95~0.99** — 알고리즘 다양화보다 피처 엔지니어링이 더 효과적이었음

**클래스/부분집단 불균형 처리 전수 검증**: scale_pos_weight, SMOTE, 언더샘플링, 부분집단 가중치, IVF/DI 분리학습 등 — **전부 손해 또는 무효로 확인** (재가중 강도에 비례해 단조적으로 악화)

## 7. 앙상블

| 단계 | 방식 | OOF AUC | 실제 제출 |
|---|---|---|---|
| 1차 | 우리팀 3종 배깅 + 팀원 v5, 가중평균(0.6:0.4) | 0.74071 | 0.7418495884 (6위) |
| 2차 | 12개 모델 → **ECDF 랭크변환 + 선형회귀/Ridge 메타러너** | **0.74098** | **0.7421890122 (3위)** |

**ECDF + 선형회귀 메타러너 핵심 차이점**: fold-train에서만 ECDF 적합(누수 없음) → 백분위 정규화 → 선형회귀로 결합(가중평균과 달리 **음수 계수 허용**, 모델간 공유 노이즈 상쇄 가능). Ridge(alpha 0~30) 재검증으로 결과 안정성 확인.

**Greedy Ensemble Selection**: 한 번에 하나씩, 개선될 때만 채택하는 보수적 방식으로 별도 교차검증 — 큰 후보 풀에서 발생하는 다중공선성 위험을 줄이는 대안으로 함께 검증.
