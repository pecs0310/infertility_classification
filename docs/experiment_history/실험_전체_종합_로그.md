# 난임 시술 임신 성공 예측 — 전체 실험 종합 로그

대회: DACON 난임 환자 임신 성공 여부 예측 AI 해커톤 | 평가지표: ROC-AUC (Public 100%, 최종 점수와 동일)

**점수 추이**: 0.536(버그) → 0.7400(베이스라인) → 0.74071(1차, 실제 0.74185, 6위) → **0.74098(2차, 실제 0.74219, 3위)**

이 문서는 프로젝트 시작부터 현재까지 시도한 **모든 실험을 시간순으로 통합**한 로그입니다.
(이전에 작성된 `experiment_history_v1.md`, `v2.md`, 1차/2차 보고서를 전부 포함하며, 이후 더
정밀하게 재검증되어 결론이 바뀐 항목은 최신 결론으로 갱신했습니다.)

## 2차 — 컴플라이언스 강화 + 부분집단 불균형 전수 검증 + 풍부한 피처셋 + 스태킹

### 2차-1. 대회 규칙 재확인 + 데이터 누수 컴플라이언스 점검
- DACON 규칙 페이지 직접 재확인: ROC-AUC, 외부데이터 금지, 사전학습모델 허용, 라벨/원핫 인코딩 시 test 활용 = Data Leakage
- **XGBoost 네이티브 categorical에서 `train ∪ test` 합집합으로 카테고리 구성하던 위반 발견** → train 카테고리만 사용, test 신규값은 NaN(XGBoost 결측 처리)으로 수정. 영향 행 3/90,067건, 점수 변화 없음
- **팀원 v5 파이프라인에서 동일 위반 패턴 발견**(1차-5에서 "누수 없음"으로 잘못 판단했던 부분) → train 카테고리 + 최빈값 fallback으로 수정. 영향 행 0건, 점수 변화 +0.00001(0.74042→0.74043)

### 2차-2. 환자 단위 데이터 누수 가능성 진단
- "안정적 특성 조합"이 같은 행 그룹 분석: train 내 반복 구조가 무작위 기준선보다 통계적으로 유의하게 존재(그룹크기 6: 관측 4.8% vs 무작위기준 0.9%)
- 그러나 **전체 피처 기준 완전 일치 행은 train·test 통틀어 0건** → 직접 익스플로잇(답 베끼기)은 불가능함을 확인

### 2차-3. 클래스/부분집단 불균형 처리 전수 재검증 (전부 손해 또는 무효로 재확인)

| `is_unbalance=True` | -0.00019 |
| SMOTE(SMOTENC) 오버샘플링 | **-0.02769(치명적), 속도도 20배 느림** |

| 나이×난자출처 인터랙션 피처 | +0.00002(무효) |
| 환자(추정)그룹 target encoding | **-0.156(치명적), 부스팅 순차학습과 구조적 충돌로 확인** |

| "구조적 변형" 스태킹(routed+eggweight+baseline 블렌딩) | 가벼운 파라미터에서 +0.00044처럼 보였으나, 실제 robust 파라미터로는 +0.00010(노이즈) — **수확 없음으로 최종 결론** |

### 2차-6. 풍부한 피처 엔지니어링 설계 (돌파구)
기존 `-1`/`해당없음` 일괄 대치 대신:
- **Keep-NaN**: DI 구조적 결측을 진짜 NaN으로 유지 + `_missing`/`_not_applicable_DI` 플래그, 트리 네이티브 결측분기 활용
- **아웃라이어 클리핑** + `_high_flag` (배아/난자 개수 컬럼)
- **안전한 비율(ratio) 피처**: 분모 0/결측 시 진짜 NaN + `_available` 플래그 (기존 `/(x+1)` 방식보다 정교)
- **실효 모성 나이**: 기증난자 사용 시 환자 나이 대신 기증자 나이 중간값으로 대체, `donor_rejuvenation_gap` 동반 생성
- **시술유형 토큰화**: `특정 시술 유형` 텍스트에서 IVF/ICSI/IUI/FER/BLASTOCYST 등 하위 토큰 분리
- **네이티브 categorical**: `pd.Categorical` dtype 사용(train 카테고리만, test 신규값은 sentinel)

→ 이 피처셋 + **CatBoost 7시드 배깅(`donor_cat`)**: **OOF 0.74076, 단독 모델 역대 최고**
→ 클리핑 유무 단독 비교: +0.00009(노이즈, 유지하되 손해 아님 확인)
→ `donor_cat` 3시드→7시드: +0.00003뿐 — 멀티시드 배깅의 수확체감 구간 진입 확인

### 2차-7. ECDF 랭크변환 + 선형회귀/Ridge 메타러너 스태킹
- 각 fold의 train 부분에서만 ECDF(경험적 누적분포함수) 적합 → 전 모델 예측을 백분위로 정규화(fold-safe)
- **선형회귀로 결합(음수 계수 허용)** — 기존 가중평균(0~1, 합=1 제약)보다 유연
- 전체 12개 후보(donor_cat, v5_ensemble, 10seed_bagged, alpha_cat, xgboost_bagged, catboost_bagged, feature_subspace, xgb_rankpairwise, lgbm_lambdarank, lgbm_keepnan, mlp, lgbm_spw2_richfeat) → **OOF 0.74098**
- Ridge(alpha 0~30) 재검증: 전 구간 0.74097~0.74098로 안정적 — 결과가 다중공선성에 의한 우연이 아님을 확인

### 2차-8. 최종 결과
**OOF 0.74098 → 실제 제출 0.7421890122 (Public 리더보드 3위)**

### 2차-9. Pseudo-labeling 재탐구 (진행 중, 제출 보류)
- Phase0 exp15와 달리 **절대 임계값(0.9+) 대신 상대적 퍼센타일(상위/하위 3%)** 사용
- fold-safe 검증(가짜 라벨은 학습 fold에만, 검증 fold엔 절대 포함 안 함)
- 단일시드 결과: baseline 0.74047 → +pseudo 0.74076 (**+0.00029**, 노이즈의 2.6배)
- **test 행(피처)이 학습에 직접 포함되는 기법이라 규칙 회색지대로 판단** → 멀티시드 재검증 진행 중 + 대회 운영진에게 허용 여부 별도 문의, **답변 전까지 제출 보류**

## 효과 없음으로 최종 확정된 것 (재시도 불필요)

- 명시적 교차/비율/클러스터링 피처, 결측패턴 조합, 타겟인코딩(단순) — 전부 노이즈~약한 손해
- 단순 모델 다양화(XGBoost/CatBoost/로지스틱/MLP/ExtraTrees/TabTransformer/EBM/lambdarank/rank:pairwise) — 전부 상관계수 0.96+ 아니면 단독성능 너무 낮음
- 클래스 불균형 처리 전 계열(scale_pos_weight, is_unbalance, SMOTE, 언더샘플링, 부분집단 가중치) — 가중치 강도에 비례해서 단조적으로 손해
- 환자그룹 target encoding — 치명적 손해(부스팅 학습 구조와 충돌)
- IVF/DI 분리 학습, 나이대 분리 서브모델 — 통합모델이 항상 더 좋음
- 진짜 Stacking 메타러너(로지스틱/얕은모델) — 단순 가중평균보다 못함
- Feature-subspace bagging — 개별 디커럴레이션되나 배깅하면 사라짐
- 절대임계값 기반 Pseudo-labeling — 확신 표본 자체가 없음

## 효과가 있었던 것

- **멀티시드 풀파이프라인 배깅** — 모든 모델에서 +0.0003~0.0004, 가장 신뢰도 높은 단일 레버
- **풍부한 피처 엔지니어링(Keep-NaN+안전비율+실효나이) + CatBoost** — 역대 최고 단독 모델
- **ECDF 랭크변환 + 선형회귀/Ridge 메타러너** — 기존 가중평균보다 유연한 결합으로 추가 개선
- (탐구 중) **상대적 퍼센타일 기반 Pseudo-labeling** — 단일시드 신호 확인, 멀티시드/규칙 검증 진행 중
