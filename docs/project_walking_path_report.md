# Infertility Classification 프로젝트 시행착오 및 극복 보고서 (Walking Path)

본 보고서는 프로젝트의 모든 진화 과정을 **EDA, 전처리, 피처 엔지니어링, 베이스라인 구축, 파생 피처, 모델링, 앙상블**의 7개 핵심 파트로 분류하여, 각 단계에서 겪은 기술적 한계(시행착오)와 이를 극복해 낸 구체적인 솔루션을 기록한 종합 트랙 리포트입니다.

---

## 📊 파트별 핵심 진화 요약
```
[EDA] ─────────► 임상 단계별 결측 규칙 발견 & 난자 출처 영향력 파악
   │
[전처리] ──────► 단순 대치 탈피, 이식 완료 주기 대상 타겟 보간 & MLP 원핫인코딩 다원화
   │
[피처엔지니어링] ► Ablation Study를 통한 피처 슬림화 & DACON 누수 방지 Sentinel 매핑
   │
[베이스라인] ──► Stratified 5-Fold 구축 및 난수 차단을 위한 5-Seed 배깅 고정
   │
[파생 피처] ───► clinical_state(7단계 상태), 이원 결측 플래그, 실효 모성 나이 설계
   │
[모델링] ──────► GBDT 3종(LGB/XGB/Cat) + MLP 개편 + Random Forest/Extra Trees 신규 주입
   │
[앙상블] ──────► ECDF 랭크 스케일 보정 -> Nelder-Mead 최적화 -> ECDF Ridge Stacking 노이즈 차감
```

---

## 1. EDA (탐색적 데이터 분석)
* **시행착오**:
  - 초기에는 타겟 변수(`임신 성공 여부`)와 개별 변수 간의 단순 피어슨 상관계수 분석에 의존했으나, 전체 데이터셋의 절반 가까이 차지하는 결측치(`NaN`) 때문에 변수 간의 선형 관계 및 영향도가 심하게 왜곡되어 표기되었습니다.
* **극복 및 발견**:
  1. **결측치 구조 분석**: 결측이 발생하는 주기가 무작위가 아니며, 시술 방식(시술 유형, 특정 시술 유형, 동결 여부 등)에 따라 구조적으로 발생하거나(MAR), 특정 단계의 시술 실패(MNAR)로 인해 발생한다는 규칙을 발견했습니다.
  2. **나이 & 난자 출처 상호작용**: 환자 본인의 난자를 쓸 때와 기증받은 난자를 쓸 때 나이에 따른 임신 성공률의 분포가 완전히 다른 패턴을 보임을 파악했습니다.
  3. **배아 이식 경과일 도메인 분석**: 이식된 날짜 데이터가 `3.0일`과 `5.0일` 부근에 강하게 군집을 이루고 있음을 발견하여, 이것이 3일배아(분할기) 및 5일배아(포배기) 이식 등 난임 임상 도메인 지식과 완벽히 일치함을 검증했습니다.

---

## 2. 전처리 (Preprocessing)
* **시행착오**:
  - 수치형 결측치를 단순히 평균값이나 일괄적으로 `-1`로 대치했을 때, GBDT 트리 모델이 결측 패턴 자체에 과적합(Overfitting)하여 로컬 검증 점수는 높으나 실제 리더보드 점수가 급락하는 현상이 발생했습니다.
* **극복 및 솔루션**:
  1. **임상 통계치 기반 타겟 보간**: 배아 이식이 실제로 완료된 주기에 한해서만 `배아 이식 경과일`을 전체 학습 데이터의 실제 중앙값인 **`3.0일`**로 정밀 보간하고, 이식이 중단된 주기는 공란으로 두어 모델에 왜곡된 정보가 들어가는 것을 차단했습니다.
  2. **기증자 정보 결측 보치**: 난자/정자 기증 주기의 누락된 나이 정보에 대해 도메인 최빈값(난자 기증자: `만31-35세`, 정자 기증자: `만21-25세`)으로 논리적 대치를 적용했습니다.
  3. **MLP 전처리 파이프라인 분리 (v18)**: MLP 모델 입력 시 범주형 변수를 수치 크기 관계가 있는 것처럼 넘기던 `OrdinalEncoder`를 전면 제거하고, **One-Hot Encoding + SimpleImputer(중앙값 대치) + StandardScaler** 전용 전처리 파이프라인을 다원화하여 MLP 성능을 정상 궤도로 올렸습니다.

---

## 3. 피처 엔지니어링 (Feature Engineering)
* **시행착오**:
  - 논문 분석 등을 토대로 유래한 대량의 파생 피처들을 무작위로 계속 추가하여 피처 수가 95개 이상(v11)으로 늘어났으나, 유의미하지 않은 노이즈 피처가 다수 섞여 오히려 검증(OOF) 점수가 정체되는 병목 현상이 발생했습니다.
* **극복 및 솔루션**:
  1. **Ablation Study (피처 소거 실험) 수행**:
    - 각 파생 피처 그룹을 제거하면서 OOF AUC의 변동을 측정하는 소거법 테스트를 거쳐, 성능 저하를 일으키는 불필요 피처를 과감히 삭제하고 최정예 피처(81개)로 슬림화에 성공했습니다(v12).
  2. **DACON 데이터 누수(Data Leakage) 규정 완벽 준수**:
    - 범주형 타겟 인코딩이나 카테고리 풀(Pool) 구성 시, 반드시 각 Fold의 Train 데이터셋만을 기준으로 fit하고 Val/Test 데이터셋은 transform만 하도록 강제했습니다.
    - Test 데이터에만 단독으로 나타나는 새로운 미관측 범주나 결측은 **`정보없음 (SENTINEL)`** 카테고리로 맵핑 고정하여 미래 데이터 정보 주입을 원천 차단했습니다.

## 4. 베이스라인 구축 (Baseline)
* **시행착오**:
  - 초기 단일 시드(Seed) 기반 검증 시, Fold를 나눌 때 발생하는 데이터 난수 노이즈에 의해 피처 튜닝 효과가 묻혀버리거나 성능이 왜곡 판단되는 불안정함이 존재했습니다.
* **극복 및 솔루션**:
  1. **Stratified 5-Fold CV 도입**:
    - 불균형 타겟(`임신 성공 여부` 비율)을 감안하여 각 Fold마다 성공/실패 비율이 학습과 검증에 균등하게 유지되도록 계층적 5-Fold를 설계했습니다(v1).
  2. **5-Seed 멀티시드 배깅 구조 고착 (v14)**:
    - 5가지의 서로 다른 난수 시드(42, 1004, 7, 2026, 88)를 기반으로 총 25번의 CV 학습을 거치는 멀티시드 배깅 구조를 베이스라인으로 고정했습니다. 시드 평균화를 통해 모델의 예측 분산을 획득하고 난수 노이즈가 완벽하게 통제된 견고한 검증 점수를 도출했습니다.

## 5. 파생 피처 (Derived Features)
* **시행착오**:
  - 복잡한 수학식으로 연산된 수치형 파생 변수들을 단순 추가했으나, 트리 모델들이 연속형 스페이스에서 변수의 논리적인 경계면(Threshold)을 제대로 쪼개지 못하는 도메인 해석력 결핍 문제가 존재했습니다.
* **극복 및 솔루션 (도메인 피처 설계)**:
  1. **`clinical_state` (7단계 상태 변수)**: 환자의 주기 도달 상태를 계층형 범주화(IUI 주기, Fresh 채취 취소, Fresh 이식 완료, Frozen 이식 완료 등)하여 모델이 주기 경로를 빠르게 파악하도록 이정표 역할을 구현했습니다.
  2. **이원 결측 플래그 (`_not_applicable` vs `_failed_or_missing`)**: 단순히 NaN 여부만 넘겨주는 대신 구조상 결측과 실패로 인한 결측을 논리적으로 분리 생성했습니다.
  3. **클리핑 및 플래그**: 수집 난자 수, 배아 수 등 극단적인 아웃라이어로 발생하는 모델 노이즈를 막기 위해 상한선(Clip)을 설정하고 고수치 이진 플래그(`{col}_high_flag`)를 병렬 주입했습니다.
  4. **6대 임상 비율 피처**: 수정율, 배아 이용률, oocyte 활용률, 해동배아이식률 등을 단순 연산하되 분모가 0이 되는 오류를 우회하는 `safe_ratio` 로직을 구축했습니다.
  5. **실효 모성 나이 (`effective_maternal_age_mid`) & 회춘 갭 (`donor_rejuvenation_gap`)**: 난자 기증 제공 주기 시 실제 나이를 보정해주고 젊은 난자 주입에 따른 나이 극복 갭 수치를 수치 피처로 환산 제공하여 핵심적인 생물학적 동력을 전달했습니다.

## 6. 모델링 (Modeling)
* **시행착오**:
  - LightGBM 단일 모델 성능 최적화에만 매달렸으나, 리더보드 특정 영역 이상 도달 시 LightGBM의 트리 분기 예측 스페이스 편향으로 인해 성능의 한계에 부딪혔습니다.
* **극복 및 솔루션**:
  1. **GBDT 3대장(LGBM / XGBoost / CatBoost) 연합 구성**:
    - LightGBM 외에 정교한 레벨별 트리 분기를 사용하는 XGBoost, 그리고 범주형 변수의 네이티브 정수 인코딩과 순서 결합에 압도적인 성능을 보이는 CatBoost를 주 모델로 적극 기용했습니다.
  2. **MLP 신경망 모델 튜닝**:
    - 트리 기반 모델과 학습 수학적 기조가 완전히 다른 다층 퍼셉트론(MLP)을 주입해 앙상블 조합 다양성을 보완했습니다.
  3. **RF / ET 아키텍처 추가 (v18)**:
    - 부스팅 계열 간의 예측 수렴 한계를 극복하기 위해, 배깅 기반의 분산 결정 모델인 **Random Forest(rf)** 및 **Extra Trees(et)**를 추가 탑재하여 모델링 다양성을 극대화했습니다.

---

## 7. 앙상블 (Ensemble)
* **시행착오**:
  - 모델들의 원본 예측 확률을 직접 평균(Soft Voting) 내어 앙상블을 진행했으나, 모델마다 가지는 확률 분포와 쏠림 스케일이 달라 상대적으로 스케일이 왜곡된 모델에 최종 점수가 동화(왜곡)되는 한계를 겪었습니다.
* **극복 및 솔루션**:
  1. **ECDF(경험적 누적분포함수) Rank Blending (v12)**:
    - 모든 모델의 출력을 ECDF를 활용해 0~1 사이의 백분위수 랭크 스케일로 통일하고 결합을 수행해 스케일 왜곡 문제를 근본적으로 격리했습니다.
  2. **Nelder-Mead 최적화 (v13)**:
    - 5-Fold OOF 랭크값을 기반으로 AUC를 최대화하는 가중치 비중(양수합=1)을 수치 최적화 알고리즘으로 자동 정밀 탐색했습니다.
  3. **ECDF Ridge Stacking (v17)**:
    - 양수 가중치 제약을 풀고 **음수(Negative) 계수를 허용하는 Ridge 메타러너 스태킹**을 도입했습니다. 이를 통해 모델 간 공유하고 있던 오답 상관관계(다중공선성) 노이즈를 감산(`Subtraction`)하여 상쇄하고 최적의 규제(alpha=100.0)로 예측 경계를 안정화하여 점수 극대화를 실현했습니다.
